#  Avito Seller Account with AI Assistant

Личный кабинет продавца с интегрированным AI-ассистентом для улучшения объявлений на основе данных карточки товара.

##  Технологии

- **Frontend:** React 18, TypeScript, Vite
- **UI:** Mantine UI v7
- **State & Data:** Zustand, TanStack Query, React Router DOM v6
- **Backend:** Express.js (mock API)
- **AI/LLM:** Ollama (Llama 3)
- **Tools:** ESLint, Prettier, Axios

##  Требования

- Node.js v20+
- npm или pnpm
- Ollama (для работы AI-функций)

##  Быстрый старт

### 1. Установка зависимостей
```bash
git clone <repository-url>
cd avito-seller-account
npm install
```

### 2. Настройка окружения
Создайте файл `.env` в корне проекта:
```env
VITE_API_URL=http://localhost:8080
VITE_OLLAMA_URL=http://localhost:11434
```

### 3. Настройка LLM (Ollama)
1. Установите [Ollama](https://ollama.ai/)
2. Загрузите модель: `ollama pull llama3`
3. Запустите сервер: `ollama serve`
4. *(Важно)* Если возникают CORS-ошибки при запросах из браузера, запустите Ollama с разрешенными источниками:
   ```bash
   OLLAMA_ORIGINS="*" ollama serve
   ```

### 4. Запуск Backend (Mock API)
```bash
cd server
npm install
npm start
```
Сервер будет доступен на `http://localhost:8080`. Данные хранятся в памяти и сбрасываются при перезапуске.

### 5. Запуск Frontend
```bash
npm run dev
```
Приложение откроется на `http://localhost:3000`

##  Принятые самостоятельные решения

1. **UI-библиотека:** Выбран **Mantine UI v7**. Обоснование: отличная TypeScript-поддержка, встроенные хуки для тем (`useMantineColorScheme`), гибкая система сеток и современная архитектура компонентов без лишних оберток.
2. **Управление состоянием:** Использован **Zustand** вместо Redux/MobX. Обоснование: минимальный бойлерплейт, отсутствие необходимости в провайдерах, идеален для локального стейта (тема, черновики, текущее объявление).
3. **Data Fetching:** Внедрен **TanStack Query**. Обоснование: автоматическое кеширование, фоновые рефетчи, удобная инвалидация кэша после `PUT`-запроса и встроенные состояния `isLoading`/`error`.
4. **Валидация доработок:** Логика `needsRevision` вынесена в отдельную утилиту `checkRevision.ts` и дублируется на бэкенде. Это гарантирует синхронизацию между клиентским UI и серверной фильтрацией.
5. **Черновики формы:** Реализовано автосохранение в `localStorage` через кастомный хук `useLocalStorage`. При переходе на `/ads/:id/edit` данные приоритетно берутся из черновика, если он существует.
6. **Обработка AI-ответов:** Внедрен строгий парсинг текста от LLM с fallback-логикой. Цены извлекаются регулярными выражениями с валидацией диапазона (>1000₽), чтобы избежать применения абсурдных значений. Добавлен `AbortController` с таймаутом 30с для предотвращения зависания UI.
7. **Верстка карточек:** Фиксированная высота карточек (`320px`) с `flex-direction: column` и `lineClamp` для заголовков. Это полностью предотвращает "скачки" сетки при разном объеме описаний товаров.
8. **Темная тема:** Реализована через нативный `useMantineColorScheme`. Переключатель вынесен на уровень навигации страниц, состояние синхронизируется между роутами без лишних рендеров.

##  Известные ограничения
- Изображения заменены на placeholder'ы согласно ТЗ.
- Backend реализован как in-memory mock. Для production требуется подключение к реальной БД.
- Ollama требует локального запуска и ~4.7GB дискового пространства для модели `llama3`.
- Фильтрация `needsRevision` на клиенте зависит от формата ответа бэкенда; при смене схемы данных потребуется адаптация парсера.
